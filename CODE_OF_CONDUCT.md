# Code of Conduct

This Code of Conduct sets expectations for behavior in project spaces. It exists to keep collaboration productive, welcoming, and safe for everyone involved. By participating in this project, you agree to follow these guidelines.

## Our pledge

We want this project to be a welcoming, harassment-free community for everyone. This pledge applies to all contributors and participants, regardless of background or identity. We will prioritize the health of the community alongside technical outcomes.

We want this project to be a welcoming, harassment-free community for everyone.

## Expected behavior

This section lists the behaviors we want to see in issues, pull requests, discussions, and code reviews. These expectations are intentionally simple and practical. When in doubt, choose the option that reduces friction and increases clarity.


- **Respect:** Be respectful and constructive.
- **Assume good intent:** Ask clarifying questions and avoid jumping to conclusions.
- **Stay focused:** Keep discussions focused on whatâ€™s best for users of this project.
- **Protect privacy:** Do not share private calendar data, API keys, or sensitive logs.

## Unacceptable behavior

This section describes behavior that is not allowed in project spaces. It is not exhaustive, but it covers common patterns that harm collaboration or safety. Maintainers may act on both the letter and the spirit of these rules.


- **Harassment:** Harassment, hate speech, or discrimination.
- **Personal harm:** Threats, doxxing, or sharing personal information without consent.
- **Sexual content:** Sexualized language or imagery.
- **Disruption:** Deliberate disruption (spam, trolling, repeated bad-faith arguments).

## Reporting

This section explains how to report Code of Conduct concerns. Reports will be handled as privately as practical, and we will focus on keeping the community safe. Please include enough detail for maintainers to understand what happened.

If you experience or witness unacceptable behavior, please report it privately to the maintainers.


- **Preferred:** GitHub Security Advisories (private report).
- **Alternative:** Email the maintainer address listed in the repository profile.

Include:

- **What happened:** Links, screenshots, and timestamps.
- **Who was involved:** Usernames or profiles.
- **Context:** Any extra context that helps us understand the situation.

## Enforcement

This section describes what maintainers may do in response to unacceptable behavior. Actions are taken to protect the community and keep project spaces usable. The exact response will depend on severity and context.

Maintainers may remove content, block accounts, or take other actions they deem appropriate to keep the community safe.

## Scope

This section clarifies where this Code of Conduct applies. In general, it covers project spaces and contexts where someone is representing the project. If you are unsure whether a situation is in scope, report it and let maintainers decide.

This Code of Conduct applies in project spaces (issues, PRs, discussions, code reviews) and when representing the project in public.
